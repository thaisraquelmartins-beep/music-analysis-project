# 🎧 Music Streaming Analysis: User Behavior & Hypothesis Testing

## 📌 About the Project
This project explores user behavior in a music streaming platform by analyzing data from two different cities: Springfield and Shelbyville.

The main goal is to test a hypothesis about how user activity varies depending on the day of the week and location.

## 🎯 Objective
To determine whether user listening behavior differs based on:
- Day of the week
- City

## 🛠️ Tools Used
- Python
- Pandas
- Data Cleaning & Preprocessing
- Exploratory Data Analysis

## 📊 Project Steps

### 1. Data Overview
- Initial exploration of the dataset
- Understanding structure and variables

### 2. Data Preprocessing
- Renaming columns
- Handling missing values
- Removing duplicates

### 3. Hypothesis Testing
- Comparing user activity by city
- Analyzing listening patterns by day
- Validating or rejecting the hypothesis

## 🔍 Dataset Description
The dataset includes:
- `userID` – unique user identifier
- `Track` – song title
- `artist` – artist name
- `genre` – music genre
- `City` – user location
- `time` – time the track was played
- `Day` – day of the week

## 💡 Key Insights
- User behavior varies depending on the city
- Listening patterns change across different days of the week
- Data preprocessing plays a critical role in accurate analysis

## ✅ Conclusion
The hypothesis was tested using real data, allowing insights into user behavior patterns and supporting data-driven decision-making.
